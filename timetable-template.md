# timetable-data.json 更新テンプレート・手順書

このファイルは `timetable-data.json` を毎年のダイヤ改正時に更新するための手引きです。
`index.html` 側のロジックは触らず、`timetable-data.json` だけを差し替える(または
`stationSets` にキーを追加する)ことを想定しています。

## 1. 全体構造

```json
{
  "schemaVersion": 1,
  "meta": { "lastUpdated": "YYYY-MM-DD" },
  "stationSets": {
    "駅セットのキー(英数字)": {
      "displayName": "人間が読むための名前",
      "anchorStation": "起点となる駅名(下記アダプタが解釈する)",
      "source": "取得元のURLまたは説明",
      "revision": "改正年月日(表示・記録用)",
      "calendar": "weekday | saturday_holiday",
      "destLabel": { "コード": "行先の駅名", ... },
      "destNote": "コードの意味の説明文(人間向け)",
      "entries": [
        [時, 分, "行先コード", 急行フラグ(true/false)],
        ...
      ]
    }
  }
}
```

`entries` の1レコードが列車1本の発車情報です。`行先コード` は `destLabel` に
定義したキーのいずれか(無印は空文字 `""`)。急行フラグは、時刻表上で赤字・太字
など「急行」を示す表記があれば `true`、通常表記なら `false`。

## 2. 新しい駅を追加する手順

1. **データ源を探す**
   - 都営交通局(`kotsu.metro.tokyo.jp/subway/timetable/...`)は静的HTMLで直接
     取得できる駅が多い(このアプリで実績あり)。
   - 東急・東京メトロの公式サイトはJS描画のため直接取得できないことが多い。
     代わりに、駅で配布される「印刷用PDF時刻表」があれば使える(このアプリの
     白金高輪駅データはこの方式)。
   - Yahoo!路線情報など第三者の乗換案内サイトは、時刻データの二次利用を明示的に
     禁止していることが多いため使わない。

2. **PDFの場合は座標ベースで抽出する**
   数字と行先コードの位置がテキスト抽出だとズレることがあるため、
   `pdfplumber` で文字ごとの `x0`(横位置)・`top`(縦位置)・
   `non_stroking_color`(文字色)を取得し、色で急行/各停を判定するのが確実。
   おおまかな流れ:
   ```python
   import pdfplumber
   pdf = pdfplumber.open("駅名.pdf")
   page = pdf.pages[0]
   chars = page.chars
   # 1) 時刻(分)の文字: フォントサイズが大きい方(本文サイズ)
   # 2) 行先コード: フォントサイズが小さい方、時刻の少し上(topが小さい)
   # 3) 急行判定: non_stroking_color が黒(0,)以外なら赤字=急行、という具合に
   #    実際の色タプルを print して確認してから閾値を決める
   ```
   数字をグループ化する際は、同じ2桁の分(例:「19」)を構成する2文字のx0の
   差が小さい(だいたい5pt前後)ことを利用し、離れていれば別の時刻とみなす。
   行先コードは、対応する時刻グループの**最後の桁**のx0に最も近いものを採用し、
   一度使ったコードは他の時刻に再利用しない(でないと隣の時刻に誤って
   割り当たることがある)。抽出後は、コードの総数が元の凡例の並びと一致するか
   必ず突き合わせて検証すること。

3. **`stationSets` にキーを追加**
   `entries` を書き終えたら、`index.html` 側に「このキーのコードをどう経路に
   変換するか」を教えるアダプタ関数を1つ追加する必要があります(下記3参照)。
   データ(JSON)とロジック(index.html)は別なので、データだけ差し替えても
   既存駅は動き続けますが、**新しい駅を追加する場合は最初の1回だけ**
   `index.html` 側に数行のアダプタを足す必要があります。

## 3. 既存駅を年次更新する手順(ダイヤ改正のたび)

1. 同じ取得元(都営サイト/PDFなど)から最新版を再取得する
2. `entries` を丸ごと差し替える(時間帯を広げる場合はここで追加してもよい)
3. `revision` と `meta.lastUpdated` を更新する
4. `index.html` 側は変更不要(ロジックは毎年変わらない前提)
5. `sw.js` の `CACHE_NAME` の末尾番号を1つ上げる(Service Workerのキャッシュを
   確実に更新するため。これを忘れると、ユーザーの端末で古いJSONが
   表示され続けることがある)
